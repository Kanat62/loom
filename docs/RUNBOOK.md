## 3. Фаза 2 — Context Engineering (prompt-кэш)

**[РЕАЛЬНЫЙ ПРОГОН]** Два вызова подряд одной и той же роли с ОДИНАКОВЫМ
системным промптом должны показать `cache_read_input_tokens > 0` на втором:
```bash
node --env-file=.env --no-warnings -e "
import('./gateway.js').then(async (g) => {
  const messages = [
    { role: 'system', content: 'Ты тестовый ассистент. Всегда отвечай ровно: OK' },
    { role: 'user', content: 'ping' },
  ];
  await g.chat('cheap', messages, { json: false, run_id: 'cache-check-1' });
  await g.chat('cheap', messages, { json: false, run_id: 'cache-check-2' });
  console.log('готово — смотри events ниже');
});
"
node --env-file=.env --no-warnings -e "
import('./journal.js').then(j => {
  for (const e of j.listEvents({ type: 'usage' })) console.log(e.run_id, JSON.parse(e.payload).cache_read);
});
"
```
Ожидание: у события с `run_id='cache-check-2'` поле `cache_read` больше 0
(первый вызов создаёт кэш — `cache_creation` > 0 — второй его читает).
Системный промпт короткий, кэш у Anthropic обычно требует ~1024+ токенов —
если `cache_read` всё ещё 0, увеличьте текст системного промпта в этом
тесте (не критично для боевых промптов — `coder.md`/`architect.md` заведомо
крупнее порога).

Сравнение токенов по агентам (агрегат §14 ТЗ):
```bash
node --env-file=.env --no-warnings -e "
import('./journal.js').then(j => {
  // грубое агрегирование без raw SQL — адаптер намеренно не отдаёт сырой доступ
  const events = j.listEvents({ type: 'usage' });
  const byAgent = {};
  for (const e of events) {
    byAgent[e.agent] ??= { n: 0, sumIn: 0 };
    byAgent[e.agent].n++;
    byAgent[e.agent].sumIn += e.tokens_in || 0;
  }
  for (const [agent, s] of Object.entries(byAgent)) console.log(agent, 'avg tokens_in =', (s.sumIn / s.n).toFixed(0));
});
"
```

Бюджет-стоп на реальном лимите уже проверен в MOCK (`npm run budget-eval`) —
механизм идентичен для реальных вызовов, отдельного реального прогона не
требует.


3. **rm -rf / не трогает хост.** ⚠️ Выполняйте ЭТОТ шаг только после того,
   как шаг 2 подтвердил `Docker-песочница активна` — без Docker команда
   `rm -rf /` пойдёт по локальному bash-фолбэку и НЕ будет изолирована.
   ```bash
   node --env-file=.env --no-warnings evals/manual/docker-isolation-task.js
   ```

   стандартная изоляция Docker, наши флаги `--network none --memory 2g
   --cpus 2 --pids-limit 256` дополнительно ограничивают ресурсы и сеть, но
   изоляцию файловой системы даёт сам факт контейнеризации). После прогона
   убедитесь, что `C:\Users\USER\Desktop\dev\loom` и остальной хост целы.

   2. Если `data_unavailable` — прогон честно останавливается ДО архитектора,

4. Если `proceed` — обычный project-поток (architect→coordinator→live_in),
   отчёт консультанта заканчивается блоком «Данные» с источниками (§17).

Ожидание по Г§10 (не проверяется кодом — оценивайте глазами):
